# 文案与视觉风格

## 反 AI 味

AI 味通常来自四类问题：

1. 空泛：只有“优化体验”“提升效率”，没有具体对象。
2. 翻译味：句式像英文转中文，例如“我们很兴奋地宣布”。
3. 过度完整：每段都整齐总结，像报告模板。
4. 结尾套话：感谢阅读、希望有帮助、一起进步。

修法：

- 用当天发生的具体事替代抽象判断。
- 每 80-120 字给一个实物：接口名、截图、错误码、用户反馈、配置项、耗时。
- 保留合理的不完美：绕路、失败、未完成、下一步。
- 中文句子短一点，少用“因此/首先/其次/最后”。

## 小红书文案

- 标题 16-28 个中文字优先，包含具体对象。
- 开头第一句不要铺垫，直接给场景或冲突。
- 标签 3-8 个，不要堆无关热词。
- 正文适合短段落，每段只说一个点。

## 小红书图片

建议竖版 `1024x1536`，适合手机首屏。

常见卡片：

- 封面：大标题 + 一句具体副标题 + 低噪背景。
- 问题卡：错误码、日志、用户反馈或 before 状态。
- 过程卡：步骤、分支、决策树。
- 结果卡：上线结果、状态变化、数据。
- 总结卡：下一步、复盘、注意事项。

视觉关键词：

- 干净但不要像企业 PPT。
- 明确层级、适度留白、局部贴纸标注。
- 可以使用产品截图、代码片段、日志块、渲染图效果。
- 避免一整张只有渐变和大字。

## 生图质感规则

参考 baoyu-skills 的小红书图片思路：不要只写“干净、真实、好看”，必须同时指定 Style、Layout、Palette。

Style 预设：

- `notion-lab`: Notion 风格知识卡 + 产品实验室质感，白底、网格、模块化信息块。
- `study-notes`: 小红书学习笔记风格，高亮笔、便签、步骤编号、重点圈注。
- `sketch-notes`: 开发者草图笔记风格，线框图、流程箭头、代码片段、真实日志。

Layout 预设：

- `hero-dashboard`: 上方大标题，中部真实界面/设备渲染，下方三枚状态指标。
- `flow-map`: 电脑 -> HTTP 地址 -> 手机的流程地图，节点有小状态灯。
- `checklist-debug`: 检查清单 + 请求日志 + 局部标注，适合排查过程。

Palette 预设：

- `macaron-tech`: 奶油白、雾蓝、薄荷绿、少量珊瑚橙，科技但不冷。
- `warm-paper`: 暖白纸张、浅杏、墨黑、荧光黄标注，笔记感更强。
- `clean-neon`: 白底、深墨蓝、亮蓝、荧光绿状态点，适合日志和链路检查。

每张图至少包含：

- 明确的信息密度说明。
- 材质和微阴影描述。
- 真实界面、日志块或流程节点。
- 局部标注、状态点、小贴纸或便签。
- 禁止项：不要官方 logo、不要空泛渐变、不要企业 PPT、不要过度简单图标堆砌。

## 微信公众号文章

- 标题比小红书更耐读，少用夸张语气。
- 摘要不超过 120 字，直接说明文章价值。
- 正文可以更完整，但段落要短。
- 需要封面图和 `thumb_media_id` 才能走官方草稿发布。
